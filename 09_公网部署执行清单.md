# 公网部署执行清单

## 当前结论

交付包已通过本地后端、接口、页面和响应式检查，但永久公网地址必须创建在项目所有者的 Render、Cloudflare 和 GitHub 账号下。账号授权前不能生成可长期交付的 `onrender.com` 与 `pages.dev` 地址。

## 一次性发布顺序

### 1. 建立 GitHub 仓库

将 `Week6_AB监控预测交付包` 目录作为仓库根目录提交。仓库可设为 Private，但 Render 与 Cloudflare 必须获得读取权限。

```bash
git init
git add .
git commit -m "Deploy coin game agent sandbox"
git branch -M main
git remote add origin https://github.com/<owner>/<repo>.git
git push -u origin main
```

### 2. 发布 Render Backend

1. Render 选择 `New -> Blueprint`。
2. 连接上一步仓库，读取根目录 `render.yaml`。
3. 首次将 `ALLOWED_ORIGINS=*`，部署成功后记录：

```text
https://<render-service>.onrender.com
```

4. 验证 `/health` 返回 `status=ok`、`mock_only=true`。

### 3. 构建并发布 Cloudflare Pages

在 Cloudflare Pages 连接同一仓库，配置：

| 配置项 | 值 |
|---|---|
| Framework preset | None |
| Build command | `node deployment/build_cloudflare.mjs` |
| Build output directory | `deployment/cloudflare_dist` |
| `RENDER_API_URL` | 上一步最终 Render 地址 |

部署成功后记录：

```text
https://<cloudflare-project>.pages.dev
```

### 4. 收紧 CORS

回到 Render，将 `ALLOWED_ORIGINS` 从 `*` 改为最终 Pages 地址并重新部署。若同时配置自定义域名，用英文逗号分隔。

### 5. 公网自动验收

```bash
RENDER_URL=https://<render-service>.onrender.com \
PAGES_URL=https://<cloudflare-project>.pages.dev \
node deployment/public_smoke_test.mjs
```

验收范围：

- Render 健康检查与 `mock_only` 边界；
- 9 个预设场景；
- Agent、固定规则、Holdout 三组七阶段 Pipeline；
- Cloudflare 首页与 Render 地址绑定；
- 公网 CSV 下载。

只有脚本输出 `Public deployment smoke tests: 18/18 passed` 后，Pages 地址才可作为正式测试链接提交。
